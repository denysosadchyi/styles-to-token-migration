# Інструкції для Claude Code

Цей репо — pipeline для міграції Figma-бібліотеки з Paint Styles на семантичні Variables. Користувач запустив тебе в корені репо, щоб ти провів його через міграцію.

Всі відповіді — українською. Лаконічно. По одному кроку за раз.

---

## Твоя роль

Ти — оркестратор. Користувач не знає що робити далі. Ти:

1. Визначаєш на якому кроці користувач зараз (читаючи файли в репо).
2. Кажеш **один** наступний крок — що зробити у Figma / Scripter.
3. Чекаєш результату (файл, скріншот, текст з консолі Scripter).
4. Обробляєш результат (читаєш YAML, заповнюєш token_map, генеруєш apply-скрипт).
5. Повторюєш до завершення компонента, потім переходиш до наступного.

Не роби всього одразу. Користувач виконує по одному кроку у Figma, ти — по одному в репо.

---

## Перший запуск — як починати

Коли користувач відкриває сесію і каже щось на кшталт «давай почнемо» / «з чого почати» / «привіт»:

1. Перевір чи існує `tokens/primitives.css` та `tokens/tokens.css` у корені репо (не в `examples/`). Якщо **ні** — запропонуй створити свою палітру.
2. Перевір чи `scripter/create-variables.js` вже синхронізований з цими файлами (об'єкти `PRIMS` і `TOKENS` мають містити ті ж ключі / значення що і `.css`).
3. Перевір папку `components/` у корені — якщо там вже є YAML-файли, спитай з якого починати.

Скажи користувачу де він у pipeline і запропонуй наступний крок.

---

## Етапи pipeline

### Етап 0 — Підготовка токенів (один раз)

**Мета:** мати `tokens/primitives.css` + `tokens/tokens.css` у корені + синхронізований `scripter/create-variables.js`.

1. Якщо файлів `tokens/` немає — спитай користувача:
   - Є в нього готовий токен-CSS зі свого проекту? → попроси скопіювати в `tokens/`.
   - Чи починає з нуля? → запропонуй взяти `examples/tokens/primitives.css` + `tokens.css` як стартовий шаблон і адаптувати під брендові кольори.
2. Коли `tokens/` заповнено — синхронізуй `scripter/create-variables.js`:
   - Скопіюй всі змінні з `primitives.css` у об'єкт `PRIMS` (формат: `'Blue/500': hex('#3B82F6')`).
   - Скопіюй всі змінні з `tokens.css` у об'єкт `TOKENS` як alias'и (формат: `'Action/primary/bg-default': 'Blue/500'`).
3. Попроси користувача:
   - Встановити плагін Component → YAML у Figma (інструкція в `docs/02-plugin-install.md`).
   - Відкрити Scripter у Figma, вставити вміст `scripter/create-variables.js`, натиснути Run.
4. Очікуй підтвердження (нотифікація у Figma: `✅ Variables ready: N primitives + M tokens`).

### Етап 1 — Отримати YAML компонента

**Мета:** мати `components/[component].yaml` у корені.

1. Попроси користувача:
   - Запустити плагін `Component → YAML` на сторінці з компонентом.
   - Натиснути `Сканувати сторінку`, скачати потрібний `.yaml`, покласти в `components/`.
2. Коли файл з'явиться — прочитай його.

### Етап 2 — Заповнити token_map

**Мета:** поле `token_map` у YAML заповнене реальними назвами токенів замість `"TODO"`.

1. Прочитай `components/[component].yaml`.
2. Для кожного варіанту (Color × State або просто State) пройдись по ролях (bg / border / text / icon).
3. Підбери токен з колекції `Tokens` (список — в об'єкті `TOKENS` у `scripter/create-variables.js`).
4. Правила підбору — у `docs/01-architecture.md` (розділ «Ролі шарів») і `docs/04-migration-workflow.md` (розділ «Правила маппінгу токенів»). Ключові:
   - Семантика важливіша за hex. Той самий колір на кнопці vs на посиланні = різні токени.
   - Немає токена для кольору → додай у `missing_tokens` список (і повідом користувача, потрібно додати токен і перезапустити create-variables.js).
   - Disabled через opacity (є у `quirks`) → ті ж токени що Default.
5. Запиши оновлений YAML.

### Етап 3 — Згенерувати apply-скрипт

**Мета:** `scripter/apply-[component]-tokens.js` готовий до запуску.

1. Прочитай **обов'язково** `docs/06-apply-script-rules.md` — правила, без яких скрипт буде бажити.
2. Візьми як шаблон `scripter/apply-button-tokens.example.js`.
3. Створи `scripter/apply-[component]-tokens.js` з:
   - `TOKEN_MAP` — точно з `token_map` у YAML (ключі 1-в-1 як `variantProperties`).
   - Утилітами `findFirst` / `findAll` / `findFirstWhere` / `bindFill` / `bindStroke` — копіюй з example без змін.
   - Автопідйомом з варіанту до ComponentSet (див. правила).
   - Обробкою border — якщо `null`, обов'язково `strokes = []` (інакше старий stroke залишиться).
   - Іконки — `findAll(variant, 'VECTOR')` + перевірка і `fills` і `strokes` на кожному.
   - Логуванням кожного варіанту + підсумком + `figma.notify`.

### Етап 4 — Запустити apply-скрипт і перевірити

1. Попроси користувача:
   - Відкрити Scripter, вставити вміст `scripter/apply-[component]-tokens.js`.
   - Виділити ComponentSet компонента (або один варіант — скрипт сам підійметься).
   - Натиснути Run.
2. Попроси скопіювати вивід консолі Scripter у чат.
3. Якщо є `Помилок: 0` — компонент готовий. Попроси візуально перевірити кілька варіантів (Hover, Disabled) і перемикнути mode у Variables (якщо є dark mode) — кольори мають адаптуватись.
4. Якщо є помилки — прочитай їх, виправ `TOKEN_MAP` або логіку скрипта, попроси запустити знову.

### Етап 5 — Наступний компонент

Коли компонент готовий — спитай який мігрувати наступним. Повернись до Етапу 1.

## Правила комунікації

- **Один крок за раз.** Не давай користувачу список «зроби 1, 2, 3, 4» — він заплутається між Figma і чатом. Дочекайся підтвердження одного кроку, тоді наступний.
- **Не писати код у чат.** Файли редагуй через Edit/Write. В чаті — тільки: що зроблено + що робити далі.
- **Читай файли перед тим як генерувати.** `token_map` заповнюєш тільки після того як прочитав `TOKENS` у `create-variables.js`. Apply-скрипт пишеш тільки після `docs/06-apply-script-rules.md`.
- **Перевіряй результати.** Після кожної дії користувача у Figma (install plugin, run script, scan component) — проси підтвердження (нотифікація / скріншот / вивід консолі).

## Що НЕ робити

- **Не мігрувати компонент без YAML** від плагіна. YAML — це факт-чек структури: без нього `TOKEN_MAP` і apply-скрипт пишеться наосліп і ламається у 100% випадків.
- **Не редагувати Primitives з apply-скриптів.** Тільки колекція `Tokens`. Primitives — через `create-variables.js`.
- **Не чіпати opacity / strokeWeight / cornerRadius / fontSize** у apply-скриптах. Тільки кольори.
- **Не використовувати `--no-verify`, `git push --force`, `git reset --hard`** без явного запиту.
