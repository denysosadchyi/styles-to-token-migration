# Плагін «Component → YAML» — встановлення

Плагін сканує поточну сторінку Figma, знаходить всі ComponentSet-и і генерує для кожного YAML-файл з описом структури та підказками кольорів.

---

## Що знаходиться у папці

```
figma-plugin/
├── manifest.json   — метадані плагіна (назва, точка входу)
├── code.js         — логіка плагіна (Figma plugin sandbox)
└── ui.html         — інтерфейс плагіна (iframe)
```

---

## Встановлення (локально)

### Крок 1 — Відкрити меню плагінів у Figma

В Figma Desktop або Figma Web:

```
Головне меню (☰) → Plugins → Development → Import plugin from manifest...
```

### Крок 2 — Вибрати manifest.json

У діалозі вибрати файл:

```
figma-plugin/manifest.json
```

### Крок 3 — Перевірити встановлення

Плагін з'явиться в:

```
Головне меню → Plugins → Development → Component → YAML
```

> Плагін встановлюється **локально** — тільки для вас, на цьому комп'ютері.
> Для інших членів команди — повторити ці ж кроки.

---

## Оновлення плагіна

Якщо `code.js` або `ui.html` були змінені:

```
Plugins → Development → Component → YAML → (правий клік) → Edit
```

Або просто закрити і знову відкрити плагін — Figma підхоплює зміни файлу автоматично при наступному запуску.

---

## Структура `manifest.json`

```json
{
  "name": "Component → YAML",
  "id": "component-to-yaml",
  "api": "1.0.0",
  "main": "code.js",
  "ui": "ui.html",
  "editorType": ["figma"]
}
```

| Поле | Значення | Пояснення |
|------|----------|-----------|
| `name` | Component → YAML | Назва у меню Figma |
| `id` | component-to-yaml | Унікальний ID (не змінювати після першого встановлення) |
| `api` | 1.0.0 | Версія Figma Plugin API |
| `main` | code.js | Логіка в пісочниці Figma |
| `ui` | ui.html | Інтерфейс у iframe |
| `editorType` | ["figma"] | Тільки для редактора Figma |

---

## Вимоги

- Figma Desktop або Figma у браузері (Web)
- Права на редагування файлу (viewer не може запускати плагіни)
- Node.js / npm **не потрібні** — плагін запускається напряму у Figma

---

## Troubleshooting

**Плагін не з'являється в меню**
→ Перевір, що обраний саме `manifest.json`, а не `code.js` чи `ui.html`

**"Plugin could not be loaded"**
→ Перевір, що всі три файли (`manifest.json`, `code.js`, `ui.html`) знаходяться в одній папці

**Плагін запускається але нічого не відбувається**
→ Переконайся, що на поточній сторінці є хоча б один ComponentSet (не просто Component)

**Помилка при скануванні конкретного компонента**
→ Плагін пропустить проблемний компонент і виведе його назву в списку помилок в UI. Решта компонентів продовжать скануватись.
